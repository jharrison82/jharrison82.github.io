In this chapter, we run through the typical website setup for a B2B investment manager and cover a few requirements for implementing the digital marketing infrastructure.

Content

Most websites consist of content grouped within four main sections:

  • Products — information about available products & strategies
  • Insights — regularly published blogs, videos and webinars
  • Firm information — history, investment philosophy & process, people, press releases, job postings, and contact details
  • Resources — tools, educational materials, and fund literature

Investor type & location

Before they can access any content, a website visitor is usually asked to select their investor type and/or location from a list of options. These selections can’t be verified, so lengthy disclaimers are typically added which must also be accepted.

While this can be a rather blunt and off-putting experience — and should therefore be made as seamless as possible — it does allow the website to be personalised for each visitor based on their selections, using rules set in the website content management system (CMS).

A visitor’s selections are typically stored in a cookie that is set on their browser so they don’t have to reselect them each time they visit.

Investor type

For investor type, the simplest option is to offer two choices:

  • Professional investor
  • Non-professional (or retail) investor

Although this does assume all visitors must be prospective investors, those visiting for a different reason — mainly job seekers and journalists — can be grouped under non-professionals. Their access to content can be restricted to job applications and high-level firm information, while professionals can access all content and subscribe to topics of interest.

The professional investor option is commonly divided further into the two main client groups (institutional and intermediary). This can be for familiarity reasons or because the investment manager produces content specifically aimed at one group (such as educational content for advisors to use with their retail clients).

Visitor location

When an investment manager offers products in more than one country, website visitors are usually asked to set their location so available pooled funds — and in some cases specific content — can be displayed.

Permissions

As it can only be determined if a page is appropriate for a visitor after their investor type and location have been selected, this can cause problems when visitors arrive directly from search engines.

If it turns out a page can’t be shown — when a UK-based visitor lands on a US mutual fund page, for example — a message can be overlaid to explain why it can’t be shown and offer alternative options or instructions (similar to a helpful 404 error page).

A self-referential HTML attribute called hreflang can be added to pages that target a specific country to help prevent this happening:

<link rel="alternate" hreflang="en-us" href="https://www.lam.com/mutual-funds/us-equity-fund" />

Single vs multiple website domains

If an investment manager offers products in more than one country, they may choose to host multiple websites on local country domains where products are available (for example, .com, .it, or co.uk), or stick to a single domain for all visitors. Once a location has been selected, visitors can be redirected to the relevant domain or content tailored appropriately.

While there are probably SEO benefits from hosting local websites, it can create some complications for smaller digital marketing teams in their day to day work. For example, while a website CMS usually allows a page to be published across multiple domains at the same time, when it comes to distributing content via social media and email, it can be helpful to only have to share a single URL across a global audience.

Languages

Some investment managers make their website available in multiple languages, which can be particularly taxing for the marketing team

Luckily, advances in technology are making the task much easier. Tools are available that can automate translation entirely (such as Weglot) which allow manual corrections or translation rules to be added for specific terms, so they don’t need to be constantly corrected.

Usually, URLs for multilingual websites remain in the default language of the website — usually English, if it’s a global investment manager — with a two-letter ISO language code added when another language is selected:

English:

https://www.lam.com/institutional/about-us

Spanish:

https://www.lam.com/es/intermediary/about-us

This language code tells search engines that a page is available in multiple languages and makes it easy to share content with segments of a firm’s audience who have indicated a preferred language.

Personalisation

Beyond personalising the website for various investor types and locations, there are often aspirations to expand it to include a visitor’s previous browsing behaviour, subscriptions, or even data within their CRM record (such as what products they are invested in).

However, visits to B2B investment manager’s websites tend to be brief. Unlike sites like Netflix or Amazon — where people land on the homepage with a specific intention to browse for something to watch or buy — visitors usually arrive on a specific page from a search engine or email (such as a product or blog page), find what they are looking for, and leave.

When a visitor does land on the homepage, they will most likely be looking for something specific (such as comparative information) which they want to find quickly. Therefore, making sure visitors can find what they need as quickly and easily as possible should be a priority — with a clear website structure, intuitive navigation, well-presented information, and a useful search function.

There’s no harm in personalising for content preferences — I just wouldn’t expect any earth-shattering results from doing so.
